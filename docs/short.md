## 1. 職務要約

設計と実装を両立できる、シニア寄りのクラウド・インフラエンジニアです。アーキテクチャや技術選定の議論もできる一方、Terraform でリソースを書き、GCP / AWS の設定を自分で確認・変更します。「管理・調整専業」でも「一点突破型の深掘りスペシャリスト」でもなく、複雑な環境を整理しながら技術的に正しい判断で前へ進める、というゾーンが得意です。

事業の成長スピードに合わせて、クラウド基盤・開発環境・運用体制を継続的に整えていくことを得意としています。Terraform / IaC によるインフラのコード化、GCP / AWS / Azure を横断したクラウド設計・構築・運用改善、生成 AI 活用による開発効率化まで、設計から実装・運用定着まで手を動かして一気通貫で推進できます。

スタートアップや成長フェーズの組織において、開発チームが機能開発に集中できる環境を整えながら、スケールしても崩れない基盤をつくることが得意です。属人化・権限管理・コスト可視化といった「放置すると負債化しやすい領域」も、開発速度を落とさない形で整備できるのが強みです。

**直近実績：** GCP 環境の基盤整備を担当し、開発チームの運用負荷を削減。クラウド費用の最適化で新規投資余力を確保しつつ（月額 10％以上削減・不要リソース約 20％削除）、IaC 化・IAM 整備・バックアップ・障害対応体制の整備を実現。

**経験規模：** 大規模通信インフラ（ソフトバンク）、約 200 サービス規模の AWS 移行推進（BIGLOBE・CCoE）、IoT シェアリングサービスのクラウド基盤（Luup）、成長フェーズの Web サービス基盤支援（フリーランス）。

Claude Code / GitHub Copilot / Cursor など AI 開発ツールを日常的に活用しており、新技術への適応と開発効率化にも積極的に取り組んでいます。非エンジニアや経営層とのコミュニケーションにも対応でき、少人数体制でもスピード感を持って動ける点が特徴です。

## 2. コアスキル・強み

**【主軸 / 直近実績あり】**

- **GCP クラウド基盤 設計・構築・改善：** 新規環境の設計・構築から、開発速度を支える既存環境の継続的改善（IaC 化・IAM 整理・監視・コスト可視化・障害復旧体制整備）まで対応。直近案件でクラウド費用を最適化し新規投資余力を確保（月額 10％以上・不要リソース 20％削除）
- **IaC / Terraform：** Terraform を用いた新規リソース作成・既存コード修正・インフラのコード化に対応。属人化解消・スケーラブルな基盤づくりを推進（Luup での実務経験あり）
- **生成 AI / 開発効率化：** Azure OpenAI / Dify / RAG を活用した社内業務効率化ツールの設計・導入。Claude Code / GitHub Copilot / Cursor などを日常的に活用し、開発生産性向上への取り組みを継続

**【対応可能 / 実務経験あり】**

- **AWS 設計・構築・運用改善：** Luup での実務構築・開発（S3, Lambda, Kinesis, CloudWatch, SNS 等）、BIGLOBE での CCoE 活動・約 200 サービスの AWS 移行推進・設計レビュー・ガイドライン整備を経験
- **IAM / セキュリティ整備：** 開発スピードを損なわない最小権限設計・権限棚卸し・付与削除フロー整備（GCP・AWS・Azure 対応）
- **DB 負荷対策：** MariaDB / Cloud SQL / RDS を対象に、スロークエリ調査、インデックス整理、接続数・負荷調査、レプリカ / フェイルオーバー方針検討
- **クラウドコスト最適化：** 不要リソース整理・リソースサイズ適正化による新規投資余力の確保、月次モニタリング設計、経営層への方針説明

**【付加価値】**

- **推進力・技術発信：** 非エンジニアや経営層にも伝わる資料・手順書を作成し、関係部署を巻き込みながら運用定着まで伴走。Luup および BIGLOBE にて TechBlog 立ち上げ・編集長として技術発信を推進
