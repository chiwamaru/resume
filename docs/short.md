## 1. 職務要約

クラウド基盤の設計・構築から運用改善まで、技術判断を持ちながら自ら手を動かして推進できるシニアインフラエンジニアです。Terraform / IaC によるインフラのコード化、GCP / AWS / Azure を横断したクラウド設計・構築・運用改善、生成AI活用まで、設計意図を持って実装し、運用定着まで一気通貫で対応します。

スタートアップや成長フェーズの組織において、「何をどの構成で構築するか」の技術判断から実装・定着まで担えることが強みです。スケールしても崩れない基盤設計と、それを実現する実装力を兼ね備えています。属人化・権限管理・コスト可視化といった「放置すると負債化しやすい領域」も、開発速度を落とさない設計・実装で対処します。

**直近実績：** GCP環境の基盤整備を担当し、Cloud Run / Cloud SQL を中心に構成の見直し・IaC整備・IAM最小権限設計・バックアップ設計・監視体制の構築を実施。クラウド費用の月額10%以上削減・不要リソース約20%削除を実現。

**経験規模：** 大規模通信インフラ（ソフトバンク）、約200サービス規模のAWS移行推進（BIGLOBE・CCoE）、IoTシェアリングサービスのクラウド基盤（Luup）、成長フェーズのWebサービス基盤支援（フリーランス）。

Claude Code / GitHub Copilot / Cursor など AI開発ツールを日常的に活用しており、新技術への適応と開発効率化にも積極的に取り組んでいます。技術的な設計判断・選択根拠を経営層や非エンジニアにも明確に伝えられる点と、少人数体制でもスピード感を持って動ける点が特徴です。

## 2. コアスキル・強み

**【主軸 / 直近実績あり】**

- **GCP クラウド基盤 設計・構築・改善：** 構成の技術選択・判断も含め、新規環境の設計・構築から既存環境の継続的改善まで自ら実装・推進。IaC化・IAM最小権限設計・監視・コスト可視化・障害復旧体制を実装。直近案件でクラウド費用を最適化し新規投資余力を確保（月額10%以上・不要リソース20%削除）
- **IaC / Terraform：** Terraform を用いた新規リソース作成・既存コード修正・インフラのコード化を自ら担当。属人化解消・スケーラブルな基盤づくりを実装で推進（Luupでの実務経験あり）
- **生成AI / 開発効率化：** Azure OpenAI / Dify / RAG を活用した社内業務効率化ツールの設計・実装・導入まで担当。Claude Code / GitHub Copilot / Cursor などを日常的に活用し、開発生産性向上を実践

**【実務経験 / 即戦力】**

- **AWS 設計・構築・運用改善：** Luupでの実務構築・開発（S3, Lambda, Kinesis, CloudWatch, SNS等）、BIGLOBEでのCCoE活動・約200サービスのAWS移行推進・設計レビュー・ガイドライン整備を経験
- **IAM / セキュリティ整備：** 開発スピードを損なわない最小権限設計・権限棚卸し・付与削除フロー整備（GCP・AWS・Azure対応）
- **DB負荷対策：** MariaDB / Cloud SQL / RDS を対象に、スロークエリ調査、インデックス整理、接続数・負荷調査、レプリカ / フェイルオーバー構成の比較・方針策定を経験
- **クラウドコスト最適化：** 不要リソース整理・リソースサイズ適正化による新規投資余力の確保、月次モニタリング設計、経営層への方針説明

**【横断的強み】**

- **技術判断の言語化・推進力：** 設計選択の根拠や技術方針を、非エンジニア・経営層にも明確に説明できる。関係部署を巻き込んで運用定着まで主導するファシリテーション力。LuupおよびBIGLOBEにてTechBlog立ち上げ・編集長として技術発信を推進

### ■ [業務委託] GCPクラウド運用最適化

| | |
|:---|:---|
| **期間** | 2024年10月〜 |
| **技術スタック** | GCP, Cloud Run, Cloud SQL, Cloud Storage, Cloud Build, Pub/Sub, Workflows, IAM, Cloud Monitoring, BigQuery, Artifact Registry, Secret Manager, Cloud Scheduler, Firebase |
| **役割** | クラウド構成設計・運用改善、SRE、IAM最小権限設計、バックアップ設計、監視・コスト可視化、障害復旧体制構築 |

#### 概要・成果

建設業向けWebサービスを提供する企業にて、GCP基盤の継続的改善を担当。Cloud Run のスケーリング設定・Cloud SQL のインスタンスサイズとバックアップ戦略・IAM の最小権限設計など、構成ごとに技術選択の根拠を持ちながら改善を設計・実装しました。

> クラウド費用の最適化で **新規投資余力を確保**（月額10%以上削減）／ 不要リソース **約20%削除** ／ IAM最小権限化 ／ バックアップ・障害対応体制構築

- クラウド費用を最適化し、新規機能開発・拡張への投資余力を確保（月額10%以上）
- 不要リソースを約20%削除し、管理対象を整理・開発チームの認知負荷を軽減
- IAM権限の棚卸しと最小権限設計を自ら実施し、開発スピードを維持しながらリスクを低減
- バックアップ戦略を設計し、障害時の迅速な復旧が可能な体制を構築
- 障害復旧手順書・復旧訓練を実施し、属人的な障害対応を誰でも動ける体制へ改善

| | |
|:---|:---|

| **技術スタック** | MariaDB, さくらのクラウド, Webサーバー, PHP, syslog, ログ監視 |
| **役割** | DB負荷調査、スロークエリ調査、運用改善、構成案比較、手順書整備 |

#### 概要・成果

全国にある自動車ディーラーが利用する業務システム / Webアプリケーション基盤において、MariaDBを中心にDB負荷の原因調査・構成比較・改善方針の策定まで自ら担当。Webサーバー / PHP / DB の同時処理数を横断的に分析し、ボトルネックを特定しました。

- MariaDB 5台構成のレスポンス遅延・接続集中を調査し、原因と対処方針を特定・整理
- スロークエリ・インデックス・max_connections・Web / PHP / DB の同時処理数を横断分析
- 土日のアクセス集中を想定した設定見直し方針と運用手順を策定
- レプリカ構成・フェイルオーバー・プロキシ構成を比較し、スケールアップの選択肢を提示
- 監視項目・ログ確認手順・切り戻し手順を整備し、障害対応の属人化を解消
