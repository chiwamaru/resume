## 1. 職務要約

事業の成長スピードに合わせて、クラウド基盤・開発環境・運用体制を継続的に整えていくことを得意としています。Terraform / IaC によるインフラのコード化、GCP / AWS / Azure を横断したクラウド設計・構築・運用改善、生成AI活用による開発効率化まで、設計から実装・運用定着まで手を動かして一気通貫で推進できます。

スタートアップや成長フェーズの組織において、開発チームが機能開発に集中できる環境を整えながら、スケールしても崩れない基盤をつくることが得意です。属人化・権限管理・コスト可視化といった「放置すると負債化しやすい領域」も、開発速度を落とさない形で整備できるのが強みです。

**直近実績：** GCP環境の基盤整備を担当し、開発チームの運用負荷を削減。クラウド費用の最適化で新規投資余力を確保しつつ（月額10%以上削減・不要リソース約20%削除）、IaC化・IAM整備・バックアップ・障害対応体制の整備を実現。

**経験規模：** 大規模通信インフラ（ソフトバンク）、約200サービス規模のAWS移行推進（BIGLOBE・CCoE）、IoTシェアリングサービスのクラウド基盤（Luup）、成長フェーズのWebサービス基盤支援（フリーランス）。

Claude Code / GitHub Copilot / Cursor など AI開発ツールを日常的に活用しており、新技術への適応と開発効率化にも積極的に取り組んでいます。非エンジニアや経営層とのコミュニケーションにも対応でき、少人数体制でもスピード感を持って動ける点が特徴です。

## 2. コアスキル・強み

**【主軸 / 直近実績あり】**

- **GCP クラウド基盤 設計・構築・改善：** 新規環境の設計・構築から、開発速度を支える既存環境の継続的改善（IaC化・IAM整理・監視・コスト可視化・障害復旧体制整備）まで対応。直近案件でクラウド費用を最適化し新規投資余力を確保（月額10%以上・不要リソース20%削除）
- **IaC / Terraform：** Terraform を用いた新規リソース作成・既存コード修正・インフラのコード化に対応。属人化解消・スケーラブルな基盤づくりを推進（Luupでの実務経験あり）
- **生成AI / 開発効率化：** Azure OpenAI / Dify / RAG を活用した社内業務効率化ツールの設計・導入。Claude Code / GitHub Copilot / Cursor などを日常的に活用し、開発生産性向上への取り組みを継続

**【対応可能 / 実務経験あり】**

- **AWS 設計・構築・運用改善：** Luupでの実務構築・開発（S3, Lambda, Kinesis, CloudWatch, SNS等）、BIGLOBEでのCCoE活動・約200サービスのAWS移行推進・設計レビュー・ガイドライン整備を経験
- **IAM / セキュリティ整備：** 開発スピードを損なわない最小権限設計・権限棚卸し・付与削除フロー整備（GCP・AWS・Azure対応）
- **DB負荷対策：** MariaDB / Cloud SQL / RDS を対象に、スロークエリ調査、インデックス整理、接続数・負荷調査、レプリカ / フェイルオーバー方針検討
- **クラウドコスト最適化：** 不要リソース整理・リソースサイズ適正化による新規投資余力の確保、月次モニタリング設計、経営層への方針説明

**【付加価値】**

- **推進力・技術発信：** 非エンジニアや経営層にも伝わる資料・手順書を作成し、関係部署を巻き込みながら運用定着まで伴走。LuupおよびBIGLOBEにてTechBlog立ち上げ・編集長として技術発信を推進

## 4. フリーランスとしての支援実績

### ■ [業務委託] GCPクラウド運用最適化

| | |
|:---|:---|
| **期間** | 2024年10月〜 |
| **技術スタック** | GCP, Cloud Run, Cloud SQL, Cloud Storage, Cloud Build, Pub/Sub, Workflows, IAM, Cloud Monitoring, BigQuery, Artifact Registry, Secret Manager, Cloud Scheduler, Firebase |
| **役割** | クラウド運用改善、SRE、IAM整理、バックアップ設計、監視・コスト可視化、障害復旧手順整備 |

#### 概要・成果

建設業向けWebサービスを提供する企業にて、開発チームが機能開発に集中できる環境づくりを目的に、GCP基盤の継続的改善を担当。Cloud Run / Cloud SQL / Cloud Storage を中心に、リソース利用状況の可視化、適正サイズ化、IAM整理、バックアップ・リストア手順、監視・コスト通知、障害復旧体制を整備しました。

> クラウド費用の最適化で **新規投資余力を確保**（月額10%以上削減）／ 不要リソース **約20%削除** ／ IAM最小権限化 ／ バックアップ・障害対応体制整備

- クラウド費用を最適化し、新規機能開発・拡張への投資余力を確保（月額10%以上）
- 不要リソースを約20%削除し、管理対象を整理・開発チームの認知負荷を軽減
- IAM権限の棚卸しと最小権限設計により、開発スピードを維持しながらリスクを低減
- バックアップ・リストア手順を整理し、障害時の迅速な復旧が可能な体制を整備
- 障害復旧手順書と復旧訓練により、属人的な障害対応を誰でも動ける体制へ改善
